**Task: Backend - CSV Parsing and Validation**

**Description:**

Implement the backend logic for parsing and validating CSV files containing transaction data. This will allow users to upload transaction data in bulk.

**Requirements:**

*   Implement a CSV parser to handle transaction data uploads.
*   Support standard CSV formats.
*   Validate the data in the CSV file, ensuring that it conforms to the expected data types and constraints.
*   Provide clear error handling for invalid data, including informative error messages.
*   Store the valid transaction data in the database.
*   Use Supabase for the backend implementation.
*   Use PostgreSQL for the database.

**Acceptance Criteria:**

*   The CSV parser should be able to handle various CSV formats.
*   Data validation should prevent invalid data from being stored in the database.
*   Error handling should provide informative error messages to the user.
*   Valid transaction data should be stored correctly in the database.
